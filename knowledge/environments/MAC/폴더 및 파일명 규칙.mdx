# 주의해야 할 폴더 이름 및 파일 이름

## 운영체제별 금지 문자 / 규칙 (절대 피할 것)
 - 공통 금지: 널 문자 \0 (모든 FS에서 불가)

### Windows:
 - 금지 문자: < > : " / \ | ? *
 - 끝에 점(.) 또는 공백(스페이스) 금지
 - 예약 이름(확장자 무관): CON, PRN, AUX, NUL, COM1..COM9, LPT1..LPT9 (대소문자 구분 없음)
 - 전체 경로 제한(구형 API): 260자(MAX_PATH) — 현대 Windows는 길게 쓸 수 있지만 호환성 위해 짧게 유지 권장

### macOS (APFS/HFS+):
 - POSIX처럼 /는 구분자(파일명에 불가)
 - HFS+는 내부적으로 문자 정규화(NFD)를 사용 — Unicode 정규화 주의
 - 옛날에 : 를 내부 구분자로 사용(실무상 : 사용 피함)

### Linux / 유닉스:
 - 금지: 널(\0) 및 경로 구분자 / 만 불가

---
:::tip
#### 대소문자와 충돌

Linux: 기본적으로 대소문자 구분 (case-sensitive)

Windows, macOS(기본): 대소문자 구분하지 않음 (case-insensitive)

→ 충돌 방지 위해 소문자 표준화 권장 (MyFile vs myfile 충돌 방지).
:::

:::tip
#### Unicode / Normalization

macOS(HFS+)는 **NFD(분해형)**로 파일명을 저장하는 경우가 있어, 같은 시각적 문자열이라도 다른 바이트 시퀀스가 될 수 있음.

권장: 애플리케이션 내부에서는 NFC로 정규화하여 저장하고 전송하되, macOS와 교류 시 NFD 호환성 유의.

간단 권장: 가능하면 ASCII 문자만 사용하는 것이 문제 예방에 가장 안전.
:::
